# ASOS A/B Testing Explorations

This project documents a self-guided exploration of ASOS digital experiments to understand A/B testing metrics and statistical inference. It includes both a narrative analysis (notebook) and an interactive Streamlit app so you can share the findings with hiring teams.

## Workspace layout
- `data/`: raw ASOS digital experiments dataset (CSV/Parquet and datasheet).
- `notebooks/ab_testing_analysis.ipynb`: exploratory notebook that calculates lift, conversion, revenue, and engagement metrics and saves visualization assets to `outputs/plots/`.
- `outputs/plots/`: figures generated by the notebook that the Streamlit experience will reuse.
- `streamlit_app/streamlit_app.py`: app that loads saved summaries/plots and explains the statistical takeaways with interactive controls.
- `asos-experimentation-env/`: the Python virtual environment (created inside the workspace with a similar name) you can activate before working.

## Getting started
1. Activate the virtual environment:
   ```bash
   source asos-experimentation-env/bin/activate
   ```
2. Install the required packages (notebook + Streamlit) once, e.g.:
   ```bash
   pip install -r requirements.txt
   ```
3. Run the notebook to generate the metric tables and save the plots (the exact steps will be shown inside the notebook).
4. Start the Streamlit preview:
   ```bash
   streamlit run streamlit_app/streamlit_app.py
   ```

## Deployment to Streamlit Community Cloud

1. Push this repository to a **public** GitHub repo (or a private repo if you have Streamlit Cloud access for private repos).
2. Go to [Streamlit Community Cloud](https://share.streamlit.io/) and sign in with GitHub.
3. Click **New app** and fill in:
   - **Repository**: `<your-github-username>/asos-experimentation`
   - **Branch**: `main`
   - **Main file path**: `streamlit_app/streamlit_app.py`
4. Click **Deploy**. Streamlit Cloud will install packages from `requirements.txt` and start the app.

The app reads the saved summary CSV and plot images committed to the repo, so the same visual story you built in the notebook appears in the deployed experience without re-running any analysis on the server.
